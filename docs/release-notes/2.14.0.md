# MongoDB Atlas Operator 2.14.0

> Work-in-progress release notes for the next release. Entries are accumulated
> per PR and consolidated into the release Google Doc at release time
> (see `docs/dev/release.md`).

## Warnings

## New features, improvements and bug fixes

- Added Atlas Service Account (OAuth 2.0) authentication support. Users can now
  supply `clientId` / `clientSecret` in a Connection Secret alongside (or in
  place of) the legacy `publicApiKey` / `privateApiKey` pair. The Operator's new
  `ServiceAccountToken` controller obtains and caches short-lived bearer tokens
  that all reconcilers share, so adoption does not increase load on the Atlas
  OAuth endpoint. API-key based authentication continues to work unchanged.
  See #3323.
- Every operator-adjacent Helm chart (`atlas-operator`, `atlas-advanced`,
  `atlas-basic`, `atlas-deployment`) now accepts `clientId` / `clientSecret`
  values alongside the existing API-key fields. Setting both credential types
  at the same time is rejected at chart render time. See #3329.
